CART3D Analysis Interface Module (AIM)
CART3D attributes

The following list of attributes drives the CART3D geometric definition.

  • capsAIM This attribute is a CAPS requirement to indicate the analysis the geometry representation supports.
    • capsReferenceArea [Optional: Default 1.0] This attribute may exist on any Body. Its value will be used as the Reference_Area entry in the CART3D input.
    • capsReferenceChord [Optional: Default 1.0] This attribute may exist on any Body. Its value will be used as the Reference_Length entry in the CART3D input.
    • capsReferenceX [Optional: Default 0.0] This attribute may exist on any Body. Its value will be used in the Moment_Point entry in the CART3D input.
    • capsReferenceY [Optional: Default 0.0] This attribute may exist on any Body. Its value will be used in the Moment_Point entry in the CART3D input.
    • capsReferenceZ [Optional: Default 0.0] This attribute may exist on any Body. Its value will be used in the Moment_Point entry in the CART3D input.
